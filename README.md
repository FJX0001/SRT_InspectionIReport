# SRT字幕检查工具

一个专业的Web应用程序，用于检查SRT字幕文件的格式错误、时间码问题和文本规范。完全在浏览器本地处理，确保用户数据隐私安全。

## 🔍 主要功能

- **全面错误检测**：检查时间码格式、时间顺序、时间重叠等问题
- **智能警告系统**：检测文本过长、行数过多、序号不连续等潜在问题
- **可视化时间线**：直观展示字幕在时间线上的分布
- **自动修复建议**：提供针对性的修复方案
- **格式验证**：确保符合标准SRT格式规范
- **本地处理**：所有检查在浏览器中完成，无需上传

## 🔧 使用方法

### 基本检查流程
1. **上传文件**：点击上传区域或拖拽SRT文件到指定区域
2. **粘贴内容**：或直接在文本框中粘贴SRT字幕内容
3. **开始检查**：点击"检查字幕"按钮进行分析
4. **查看结果**：查看错误、警告和建议
5. **导出修正**：使用"导出修正版"功能生成修复后的文件

### 支持的检查项目

#### 错误检测
- ❌ 时间码格式错误
- ❌ 时间顺序错误（结束时间早于开始时间）
- ❌ 时间重叠
- ❌ 序号格式错误
- ❌ 格式不完整

#### 警告检测
- ⚠️ 序号不连续
- ⚠️ 字幕文本为空
- ⚠️ 单行文本过长（>40字符）
- ⚠️ 行数过多（>2行）
- ⚠️ 时间间隔过短（<100ms）

## ⚠️ 注意事项

### 文件格式
- 仅支持标准SRT格式：`HH:MM:SS,mmm --> HH:MM:SS,mmm`
- 每字幕块包含：序号 + 时间码 + 字幕文本 + 空行
- 确保使用逗号分隔秒和毫秒部分

### 局限性
- 无法检查语义错误（如拼写、翻译）
- 不能处理非标准扩展格式
- 时间码必须符合24小时制

### 最佳实践建议
1. **时间码**：确保结束时间晚于开始时间
2. **间隔**：保持至少100ms的字幕间隔
3. **文本长度**：单行不超过40字符，每字幕不超过2行
4. **序号**：保持连续递增的序号
5. **格式**：使用标准SRT时间码格式
